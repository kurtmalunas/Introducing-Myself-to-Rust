Introduction

Rust is an open-source system level programming language for developing software that is typically written in other popular languages such as C or C++. Created by Mozilla Research employee Graydon Hoare to address issues concerning memory errors and building concurrent programs in C and C++. Some notable programs you could build with this language are CLI tools, web apps, network services, and embedded systems.

As a Java developer who mainly writes high-level codes, learning Rust may be a daunting task. For the time being, I will limit myself to using Rust in the same way that I use Java, gradually progressing to writing system-level code. However, There are benefits to using this language for a better coding experience, this includes.

  • Memory safety - Common memory-related issues are avoided before the programme is even compiled, thanks to Rust's syntax and language metaphor, as the compiler will force you to fix them before it even compiles. Null Pointers: Java vs. Rust

  • Speed - Performance is improved by Rust's zero cost abstraction and the lack of a garbage collector. If you didn't know, Java's garbage collector creates some runtime overhead that is out of our control as programmers.

  • Concurrency - By leveraging data ownership and type checking, many concurrency errors are flagged by the compiler rather than discovered at runtime.

Sure, Java is known for many criticisms. In fact, I've met some people who hate it, so why should we learn Rust when it wouldn't be surprising to say that other programming languages are superior to Java? Well, Java isn't entirely bad, just like its rivals, it has its benefits and disadvantages. Rust, on the other, hand has been regarded as the most loved programming language on Stack Overflow for many years. It is safe to say that Rust is a great language to learn given that Stack Overflow has been the most popular exchange network for programmers. Why don't we get it a try?

Concepts

  • Rustup - we installed rustup by official recommendation. It is a toolchain multiplexer responsible for installing, managing, and presenting Rust toolchains.

  • Cargo - it acts as the build system and package manager for Rust. It is a tool used to help download and manage the libraries that our code needs. This is installed automatically by rustup.

  • Rustcresponsible for translating Rust source code to machine language.

Findings

  • Variables are immutable by default for improved safety and better caching. Add mut keyword to the variable name to make it mutable.

  • Rust can specify the working space of the data types that you use. Let say I made a variable named "integer" by using the statement "let integer = 12", Rust will assume that it's a data type of i32(an integer with 32 bit of working space), we can specify this by using a semi-colon right after the variable name and specify the data type as follows "let integer: i64 = 32" giving this data type a 64 bit working space.

  • In Rust, you can create types that represents data which may be one of a few different variants within the list. Let say, you want to make an integer that can only be between 1 to 10. You can use of enums.

  • It has an organized library for errors. (https://doc.rust-lang.org/error_codes/error-index.html)

  • It has a peculiar way of making objects. An object requires a struct (a fixed set of data in a single unit) body and an impl (responsible for defining methods of a struct) block. See Line 1-29.

  • Printing variables or methods inside a string is made easier compared to Java where we have to concatenate them with each other. See line 36.

  • The language comes with a standard library and using a crate can get us more functionality into our code. They're basically external packages in Rust.

  • This language is a pain in the butt. I frequently get compile errors because of the compiler’s design. Although unpleasant, it's a good thing.

                //Input/Output library from the standard library "std"
                use std::io::{self, Read};
                //for random number generators
                use rand::{Rng, random};
                fn main() {
                  println!("Guess a number between 1 and 100");
                  println!("Enter your guess: ");
                
                  let mut guess = String::new();
                
                  //Read user input and store to the variable
                  io::stdin().read_line(&mut guess).expect("Error");
                  
                  let guess_number: i32 = guess.trim().parse().expect("Error");
                  let mut random_number = rand::thread_rng().gen_range(1..101);
                  match check(guess_number, random_number) {
                    Comparison::Less => println!("You Guessed too small"),
                    Comparison::Greater => println!("You Guessed too big"),
                    Comparison::Close => println!("Close!!"),
                    Comparison::Equal => println!("You Win!!"),
                  }
                }
                
                fn check(guess: i32, rand: i32) -> Comparison {
                    let difference = guess - rand;
                    if guess > rand && difference <= 10 {
                        Comparison::Close
                    } else if guess > rand && difference > 10 {
                        Comparison::Greater
                    } else if guess < rand && difference >= -10 {
                        Comparison::Close
                    } else if guess < rand && difference < -10 {
                      Comparison::Less
                    } else {
                        Comparison::Equal
                    } 
                }
                
                enum Comparison {
                    Less,
                    Greater,
                    Close,
                    Equal
                }

  • Guessing Game (Enhanced)

    
                    use std::io::{self, Read};
                    use rand::{Rng, random};
                    fn main() {
                      println!("Guess a number between 1 and 100");
                      println!("How many times do you want to play the game? ");
                      
                      let mut overall = String::new();
                      io::stdin().read_line(&mut overall).expect("Error");
                      let overall_number: i32 = overall.trim().parse().expect("Error");
                      let mut iteration = 0;
                      let mut correct = 0;
                      while iteration < overall_number {
                        println!("Guess the number: ");
                        let mut guess = String::new();
                        io::stdin().read_line(&mut guess).expect("Error");
                        let guess_number: i32 = guess.trim().parse().expect("Error");
                        let mut random_number = rand::thread_rng().gen_range(1..101);
                    
                        match check(guess_number, random_number) {
                        Comparison::Less => println!("You Guessed too small"),
                        Comparison::Greater => println!("You Guessed too big"),
                        Comparison::Close => println!("Close!!"),
                        Comparison::Equal => {
                            correct+=1;
                            println!("You Win!!")
                        }
                        }
                          iteration += 1;
                      }
                    
                      println!("Your correct guesses are {correct} out of {iteration}");
                    }
                    
                    fn check(guess: i32, rand: i32) -> Comparison {
                        let difference = guess - rand;
                        if guess > rand && difference <= 10 {
                            Comparison::Close
                        } else if guess > rand && difference > 10 {
                            Comparison::Greater
                        } else if guess < rand && difference >= -10 {
                            Comparison::Close
                        } else if guess < rand && difference < -10 {
                          Comparison::Less
                        } else {
                            Comparison::Equal
                        } 
                    }
                    
                    enum Comparison {
                        Less,
                        Greater,
                        Close,
                        Equal
                    }
